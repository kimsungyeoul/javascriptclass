자바스크립트의 **호이스팅(Hoisting)** 은 자바스크립트의 실행 메커니즘 중에서 매우 중요한 핵심 개념입니다. 이 개념을 정확히 이해하면 `var`, `let`, `const`, 함수 선언의 동작을 예측할 수 있습니다.

---

## 🧠 호이스팅(Hoisting)이란?

> 자바스크립트 엔진이 코드를 실행하기 전에, 스코프 안에 있는 변수·함수 선언을 먼저 등록해 두는 동작.

즉, 실제로 코드 상에서 아래에 선언했더라도, **선언만 먼저 처리되어 스코프의 맨 위에 존재하는 것처럼 보입니다**.

---

## 📌 대표적인 호이스팅 예

### 1. `var`의 호이스팅

```javascript
console.log(a); // 👉 undefined
var a = 10;
```

➡ 내부 동작처럼 바뀜:

```javascript
var a;           // 선언이 끌어올려짐
console.log(a);  // undefined
a = 10;
```

---

### 2. 함수 선언의 호이스팅

```javascript
greet(); // 👉 "Hello"

function greet() {
  console.log("Hello");
}
```

➡ 함수 전체가 끌어올려짐:

```javascript
function greet() {
  console.log("Hello");
}

greet();
```

---

### 3. `let`과 `const`는 호이스팅 O, 초기화 X

```javascript
console.log(b); // ❌ ReferenceError
let b = 20;
```

➡ `let`도 **호이스팅은 되지만**, **초기화 전에 접근하면 에러**가 납니다.

이 영역을 **TDZ (Temporal Dead Zone, 일시적 사각지대)** 라고 합니다.

---

## ✅ 변수 호이스팅 요약

| 선언 방식          | 호이스팅됨 | 초기화 전 접근 가능? | 초기값         |
| -------------- | ----- | ------------ | ----------- |
| `var`          | ✅     | ✅ 가능         | `undefined` |
| `let`, `const` | ✅     | ❌ 에러 (TDZ)   | 없음          |

---

## ✅ 함수 호이스팅 요약

| 함수 유형                               | 호이스팅 여부     | 호출 가능 시점      |
| ----------------------------------- | ----------- | ------------- |
| 함수 선언 (`function fn()`)             | ✅ 전체 끌어올림   | 선언 이전에도 호출 가능 |
| 함수 표현식 (`const fn = function() {}`) | ❌ 끌어올려지지 않음 | 선언 이후에만 호출 가능 |

---

## ⚠️ 함수 표현식은 호이스팅 안 됨

```javascript
foo(); // ❌ TypeError: foo is not a function

var foo = function() {
  console.log("Hi");
};
```

* `foo`는 `undefined` 상태에서 호출되므로 에러 발생

---

## 🧠 TDZ(Temporal Dead Zone)란?

> `let` 또는 `const` 변수는 선언은 호이스팅되지만, **초기화되기 전까지 접근이 불가능한 기간(TDZ)** 이 존재합니다.

```javascript
console.log(x); // ❌ ReferenceError
let x = 5;
```

* 변수는 스코프의 맨 위에서 호이스팅되지만,
* 선언문 이전에는 **접근할 수 없음**

---

## 🧪 실습 문제

### Q1. 다음 코드의 결과는?

```javascript
console.log(x);
var x = 10;
```

✅ 출력: `undefined`
(선언은 호이스팅, 초기화는 나중)

---

### Q2. 다음 코드의 결과는?

```javascript
console.log(y);
let y = 10;
```

❌ 오류: `ReferenceError: Cannot access 'y' before initialization`
(`let`은 TDZ로 인해 초기화 전 접근 불가)

---

## 🎯 정리

| 항목            | 호이스팅 여부 | 초기화 전 접근 가능 | 초기값       |
| ------------- | ------- | ----------- | --------- |
| `var`         | ✅       | ✅           | undefined |
| `let`/`const` | ✅       | ❌ (TDZ)     | 없음        |
| 함수 선언         | ✅       | ✅           | 함수 전체     |
| 함수 표현식        | ❌       | ❌           | 접근 불가     |

---

## 🔚 한 문장 요약

> **자바스크립트는 변수와 함수의 선언을 실행 전에 미리 메모리에 올린다. 이를 호이스팅이라 하며, 변수의 초기화 시점과는 다르다.**
