
# 🧠 자바스크립트의 **기본 타입 vs 객체 타입**

---

## 📌 1. 자바스크립트의 데이터 타입 분류

자바스크립트는 데이터를 **크게 두 가지**로 구분합니다:

| 분류                    | 타입들                                                                    |
| --------------------- | ---------------------------------------------------------------------- |
| **기본 타입** (Primitive) | `number`, `string`, `boolean`, `null`, `undefined`, `bigint`, `symbol` |
| **객체 타입** (Object)    | `object`, `array`, `function`, `Date`, `RegExp`, 사용자 정의 객체 등           |

---

## ✅ 2. 기본 타입 (Primitive Type)

### 특징

* **단일 값**을 직접 저장
* **불변값 (Immutable)**: 값을 수정할 수 없음
* **값 자체가 저장됨**
* 비교는 값으로 이루어짐

### 예제

```javascript
let a = 10;
let b = a;
b = 20;

console.log(a); // 10 (원본은 변하지 않음)
```

* `a`와 `b`는 **완전히 다른 저장 공간**에 각각의 값을 가짐

---

## ✅ 3. 객체 타입 (Object Type)

### 특징

* 여러 값을 **속성(key-value)** 형태로 저장
* **가변값 (Mutable)**: 내용을 변경할 수 있음
* 변수에는 **참조(주소)가 저장됨**
* 비교는 \*\*참조값(주소)\*\*으로 이루어짐

### 예제

```javascript
let obj1 = { name: "Alice" };
let obj2 = obj1;
obj2.name = "Bob";

console.log(obj1.name); // "Bob" (동일한 객체를 참조하므로 같이 바뀜)
```

* `obj1`과 `obj2`는 같은 객체를 가리키는 **참조 변수**

---

## 🔬 4. 내부 동작 차이

| 항목    | 기본 타입              | 객체 타입         |
| ----- | ------------------ | ------------- |
| 저장 방식 | 값 자체 저장            | 객체의 주소(참조) 저장 |
| 메모리   | 스택(Stack)에 저장      | 힙(Heap)에 저장   |
| 복사 방식 | 값 복사 (깊은 복사)       | 참조 복사 (얕은 복사) |
| 불변성   | 불변 (새로운 값을 할당해야 함) | 가변 (속성 변경 가능) |
| 비교 방식 | 값 비교 (`===`)       | 참조 비교 (`===`) |

---

## 🧪 실습 예제

### 예제 1: 기본 타입 비교

```javascript
let x = 5;
let y = 5;
console.log(x === y); // true (값 비교)
```

### 예제 2: 객체 타입 비교

```javascript
let obj1 = { id: 1 };
let obj2 = { id: 1 };
console.log(obj1 === obj2); // false (참조가 다름)
```

### 예제 3: 참조 복사의 문제점

```javascript
let userA = { name: "Tom" };
let userB = userA;
userB.name = "Jerry";

console.log(userA.name); // "Jerry" (같은 객체 참조)
```

---

## 🧠 불변 vs 가변 정리

| 구분    | 불변값 (Primitive) | 가변값 (Object)     |
| ----- | --------------- | ---------------- |
| 예시    | `const a = 1`   | `const obj = {}` |
| 수정 방식 | 새로운 값으로 **재할당** | 객체의 **속성 변경 가능** |
| 결과    | 새 값 생성          | 같은 객체의 상태 변화     |

---

## ⚠️ 주의할 점

1. 객체를 `const`로 선언해도 **속성 값은 변경 가능**함

   ```javascript
   const person = { name: "Alice" };
   person.name = "Bob"; // OK
   ```

2. 기본 타입은 `typeof`로 판별 가능

   ```javascript
   typeof 123        // "number"
   typeof "hello"    // "string"
   typeof null       // "object" ← 버그로 인해, 특별히 주의!
   typeof {}         // "object"
   ```

## 💡 자주 묻는 질문 (FAQ)

### Q1. 객체를 깊은(Deep) 복사하려면?

```javascript
const newObj = JSON.parse(JSON.stringify(obj));
```

또는 Lodash의 `cloneDeep()`을 사용하거나, 구조 분해 + 재귀 함수 등을 사용해야 함.

---

### Q2. 배열은 객체인가요?

✔️ 네, 배열은 `typeof []` → `"object"`이며, 객체 타입입니다.

```javascript
let arr = [1, 2, 3];
let copy = arr;
copy[0] = 999;
console.log(arr[0]); // 999
```

## ✅ 요약 정리

| 항목     | 기본 타입                | 객체 타입                  |
| ------ | -------------------- | ---------------------- |
| 메모리 위치 | 스택                   | 힙                      |
| 복사 방식  | 값 복사                 | 참조 복사                  |
| 불변성    | 불변 (값 수정 불가)         | 가변 (속성 수정 가능)          |
| 비교 방식  | 값으로 비교 (`===`)       | 참조로 비교 (`===`)         |
| 예시     | `1`, `"abc"`, `true` | `{}`, `[]`, `function` |
